PPE Safety Kit Object Detection

This project aims to detect PPE safety kits (Personal Protective Equipment) using the YOLO (You Only Look Once) object detection algorithm. The model can detect various types of equipment, including:

  • Hardhat
  • Mask
  • NO-Hardhat
  • NO-Mask
  • NO-Safety Vest
  • Person
  • Safety Cone
  • Safety Vest
  • Machinery
  • Vehicle

Dataset

The dataset used for training this model is the PPE detection dataset available on Roboflow. It contains images of people wearing PPE equipment in various environments.

The training set contains 2.6k images, the validation set contains 114 images, and the testing set contains 82 images. The input dimensions for each image are 640x640.


Industrial Safety Equipment Detection Project

Project Objective

The objective of this project is to ensure worker safety in industrial environments by automatically checking whether workers are wearing the required safety equipment (such as helmets, masks, and jackets). The system uses a live camera feed to monitor workers in real time.

A detection model identifies whether safety equipment is present or missing.

If a violation is detected, an email notification is sent to the concerned authority.

This helps prevent accidents and improves compliance with safety rules.

Prototype and Its Demo Status

At present, the system has been developed and tested on a local computer using a webcam.

Demo Capabilities:

  • Captures live video.
  • Detects the presence of safety gear in real time.
  • Sends email notifications when violations occur.
  • A demo video is available to illustrate how the system works.

This confirms that the concept is working successfully on a small scale.

Next Implementation: Real-Time Industrial Setup

The next step is to expand the system from a local demo to an industrial environment. This can be achieved by:

  • Using CCTV/IP cameras instead of a single webcam.
  • Running the detection software on a central server to process and monitor multiple camera feeds.
  • Sending instant alerts (buzzer, display, email) when violations occur.
  • Storing violation data with time, date, and snapshots for record-keeping.
  • Providing a dashboard for safety officers to view live status and reports.
